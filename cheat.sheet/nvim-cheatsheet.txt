PHILOSOPHY
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  Stay in Normal mode as much as possible
  Insert mode = only for typing NEW text, nothing else
  Use motions + operators instead of manual edits
  Use text objects instead of multiple motions
  Use . to repeat instead of retyping
  Use macros for repetitive multi-step edits
  Avoid arrow keys — use h/j/k/l and motions
  Think in: operator + motion, operator + text-object
  Think in: verb + noun  (d=delete, w=word → dw)
  Stop thinking "how do I move there" → think "what do I do to what object"

1. NORMAL MODE — MOVEMENT
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
BASIC
  h j k l          ← ↓ ↑ →
  w / W            next word / WORD (WORD ignores punctuation)
  b / B            previous word / WORD
  e / E            end of word / WORD
  gg               top of file
  G                bottom of file
  {number}G        go to line number  (e.g. 42G)
  %                jump to matching bracket/paren/brace

LINE NAVIGATION
  Ctrl-a           first non-blank character  [REMAPPED from ^]
  Ctrl-e           end of line                [REMAPPED from $]
  0                absolute start of line (including whitespace)
  ^                native: first non-blank character
  $                native: end of line

CHARACTER JUMPS  (extremely powerful, learn these)
  f{char}          jump forward to char on line (cursor lands on char)
  F{char}          jump backward to char on line
  t{char}          jump forward, stop before char
  T{char}          jump backward, stop before char
  ;                repeat last f/F/t/T forward
  ,                repeat last f/F/t/T backward

  EXAMPLE: f( → ; → ; → lands on each ( across the line
  COMBINE:  df(   delete from cursor up to and including (
            dt(   delete up to but not including (
            cf"   change from cursor to next "

COUNTS (combine with any motion)
  5j               move 5 lines down
  3w               jump 3 words forward
  2b               jump 2 words backward

FILE NAVIGATION
  Ctrl-d           half page down (most used — learns your rhythm)
  Ctrl-u           half page up
  Ctrl-f           full page down
  Ctrl-b           full page up
  zz               center cursor on screen
  zt               cursor to top of screen
  zb               cursor to bottom of screen

BLOCK / PARAGRAPH JUMPS
  }                jump to next empty line (next paragraph/block)
  {                jump to previous empty line
  ]]               next function or block
  [[               previous function or block

SEARCH NAVIGATION
  /pattern         search forward
  ?pattern         search backward
  n                next match
  N                previous match
  *                search word under cursor (forward)
  #                search word under cursor (backward)
  leader+nh        clear search highlights  [REMAPPED]

2. JUMP LIST — navigate your history  (extremely underrated)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  Ctrl-o           jump back in history  (works across files!)
  Ctrl-i           jump forward in history
  '.               jump to last edit position
  gd               go to definition of word under cursor
  :jumps           view full jump list

  HOW IT WORKS: every big jump (/, gg, G, %, gd, file switch) is recorded
  Ctrl-o/i walks through that list — file + exact line/column remembered
  You can edit anywhere in the history — list keeps growing

  PRO WORKFLOW: jump to check something → edit if needed → Ctrl-o back
  No need to remember line numbers or use marks for quick lookups

4. TEXT OBJECTS (nouns) — the most powerful vim concept
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  i = inner (excludes surrounding chars)
  a = around/a (includes surrounding chars)

  iw / aw          inner word / a word (with space)
  is / as          inner sentence / a sentence
  ip / ap          inner paragraph / a paragraph
  i" / a"          inside/around double quotes
  i' / a'          inside/around single quotes
  i` / a`          inside/around backticks
  i( / a(          inside/around parentheses
  i[ / a[          inside/around brackets
  i{ / a{          inside/around curly braces
  i< / a<          inside/around angle brackets
  it / at          inside/around HTML/XML tag

  KEY INSIGHT: cursor can be ANYWHERE on the line — vim finds the pair automatically

EXAMPLES (operator + text object)
  ciw              change inner word (most used — cursor anywhere on word)
  daw              delete a word including trailing space
  ci"              change text inside quotes
  ca(              change everything including the parentheses
  yi{              yank everything inside curly braces
  =ip              auto-indent inner paragraph
  >i{              indent inside curly braces
  vi"              visually select inside quotes
  di'              delete inside single quotes
